## Stage 2 — Code Quality

### ERROR Findings

#### E1 — Drift always triggers after Phase 2 (broken idempotency)

**File:** `.aai/scripts/lib/test-canon-core.mjs`, line 517
**Category:** Best Practices
**Severity:** ERROR
**Message:** `sourceHashes` are recorded from original source paths. After Phase 2 moves originals to `tests/_archive/` and deletes them, those paths no longer exist. Every subsequent run calls `hashTestSources(root, sources)` where `sources` still points to the deleted originals — all hashes return `null`. Since `null !== recordedHash`, drift is always detected. Re-run is not idempotent: the output is byte-identical only because drift blocks writes, but the behavioral contract ("re-running with no changes is idempotent — same output byte-identical modulo timestamps") is violated because drift is falsely reported every time.

**Fix:** After archiving, either:
- Update `d.sources` to point to the archived paths in `tests/_archive/`, OR
- Hash the archived copies instead of the deleted originals, OR
- Hash the canonical test file content instead of source files.

---

#### E2 — --resync reads from non-existent original paths

**File:** `.aai/scripts/lib/test-canon-core.mjs`, line 439
**Category:** Best Practices
**Severity:** ERROR
**Message:** On `--resync`, the implementation reads source contents from `d.sources` paths (the original locations). After Phase 2, these files have been moved to `tests/_archive/` and deleted. `sourceContents` will be empty, producing a canonical test file with only stubs and zero real test content. The spec requires `--resync` to "re-synthesize drifted domains from archived sources".

**Fix:** On `--resync`, use `d.archivedAt` mapping to read from `tests/_archive/` paths instead of the original `d.sources` paths. Update `d.sources` to point to the archived paths after re-synthesis.

### WARNING Findings

#### W1 — No git mv (move not tracked by git)

**File:** `.aai/scripts/lib/test-canon-core.mjs`, line 506
**Category:** Best Practices
**Severity:** WARNING
**Message:** Uses `fs.writeFileSync` + `fs.rmSync` to move originals to archive. Spec-AC-02 requires "MOVES (tracked git move)". Without `git mv`, git may not detect the rename if content similarity is below the rename threshold, losing the file history connection.

**Fix:** Use `execSync('git mv <src> <dest>')` or equivalent to ensure git tracks the rename. Fall back to fs operations only if git is unavailable.

---

#### W2 — verifyRunner only checks shebang

**File:** `.aai/scripts/lib/test-canon-core.mjs`, line 374
**Category:** Best Practices
**Severity:** WARNING
**Message:** `verifyRunner()` only checks that files start with `#!/usr/bin/env bash`. Spec-AC-04 requires verifying "the canonical suite still runs via existing runners" before archiving. No actual execution through Pester, `test-*.sh`, or `tests/skills/test-framework.sh` is performed.

**Fix:** Execute each canonical test through the project's existing runners and assert exit 0 (or expected RED for stubs). Only proceed to archive if all tests pass runner verification.

#### W3 — No canonical-to-archive back-links

**File:** `.aai/scripts/lib/test-canon-core.mjs`, line 292
**Category:** Best Practices
**Severity:** WARNING
**Message:** `renderCanonicalTest()` includes `# Sources:` lines but no `# Archive:` back-links. Spec-AC-04 requires "bidirectional back-links are present (archive → canonical and canonical → archive)". Currently only archive files get the forward `# Canonical:` pointer.

**Fix:** Add `# Archive: tests/_archive/...` lines to the canonical test file header for each archived source, so the canonical file links back to where originals are archived.
