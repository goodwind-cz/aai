### NON-BLOCKING

**W1 [WARNING] YAML-keyword slugs are written as non-string YAML (silent typing corruption).**
`SLUG_RE` admits `null`, `true`, `false`, `yes`, `off` (all ≥3 chars, lowercase) as refs, and state.mjs writes refs unquoted: `ref_id: ${ref ?? 'null'}` (state.mjs:610), `- ref_id: ${ref}` (:691), and as a mapping key `    ${ref}:` (:939). Probe-verified: `set-phase --ref null` yields `ref_id: null` which PyYAML reads as `None`; `append-run --ref true` yields a work-items key PyYAML reads as boolean `True`. check-state.mjs (regex-based) passes silently, so the corruption is invisible until a real YAML consumer (the Python assertions in the test harness, any dashboard) mis-keys the entry. `deriveSlug` CAN emit these (topic "NULL", "Yes"). Likelihood is low; failure is silent rather than loud, which is why this is a warning and not info. Fix candidate (S): reject the closed YAML-keyword set in `refFlag`, or quote keyword scalars (extend `needsQuoting`) and the work-items key. **Disposition required at closeout: remediate or promote to a follow-up ref.**

**W2 [WARNING] deriveSlug/SLUG_RE minimum-length seam.**
`deriveSlug` (allocate-doc-number.mjs:59) enforces no minimum and can emit 1-2-char slugs (`"AI"` → `ai`); such a doc's frontmatter id is then unusable as `--ref` (exit 2 under SLUG_RE min-3), though `--gate ai` still resolves it. The spec's D1 claim that min-3 excludes only "shapes deriveSlug can never emit" is inaccurate for length. Fails loud, operator can rename — but the seam should be closed on the deriveSlug side (enforce min length or pad) or documented in SPEC-0015. **Disposition required at closeout: promote to a follow-up ref.**
