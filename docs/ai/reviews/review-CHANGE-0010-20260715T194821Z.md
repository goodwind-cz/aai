### WARNING (non-blocking; per SPEC-0013 H6 each needs remediation or a decision/follow-up ref before closeout)

**W1 — `independence:` enforce silently disarmed by an invalid value.**
`.aai/scripts/state.mjs` `readIndependencePolicy()` (~line 697): `m[1] === 'enforce' ? 'enforce' : 'report-only'` maps ANY present-but-unrecognized value — typo `enforced`, quoted `"enforce"`, `Enforce` — to report-only with zero diagnostic. Failure scenario: operator writes `independence: enforced`, believes maker≠checker is enforced; same-model verdicts are merely warned forever. Spec D3 mandates the fail-open DEFAULT, but does not forbid a one-line stderr notice for a present-invalid value. Suggested disposition: follow-up (add `state: set-validation: NOTE independence value "<v>" unrecognized — treating as report-only`).

**W2 — Three independent parsers of docs-audit.yaml + presence-mode coupling.**
Guard dials are now parsed in three places with different grammars: state.mjs `/^independence:\s*([^\s#]+)/` (column-0 only; accepts `enforce#x`), pre-commit-checks.sh:212 `^[[:space:]]*doc_number_guard:[[:space:]]*enforce([[:space:]]|$)` (tolerates indentation; rejects `enforce#x`), install-pre-commit-hook.ps1:112 (same grep family for close_gate). An operator who indents the keys gets close_gate/doc_number_guard enforced but independence silently report-only. Additionally docs-audit.mjs treats the mere PRESENCE of docs/ai/docs-audit.yaml as "enforced" audit mode (docs-audit.mjs:23) — in a fresh vendored project without the file, creating it solely for `independence: enforce` side-effect-flips the docs audit into enforced mode. Not reachable in this repo (file already exists) and D3 acknowledges the naming residual, but the coupling is undocumented. Suggested disposition: follow-up (one shared note in docs-audit.yaml header + align grammars when `guards.yaml` rename lands).

### INFO (do not block)

**I1 — Single-suffix strip hardening.** `normalizeModelId()` strips exactly ONE trailing `[..]`: `claude-opus-4-8[1m][beta]` → `claude-opus-4-8[1m]` ≠ base → same-weights id evades the check; `claude-opus-4-8 [1m]` leaves a trailing space (no re-trim after strip) → unequal. Spec-mandated behavior ("one trailing bracket-suffix"); no such ids exist in history. Loop-strip + re-trim would harden.

**I2 — Implementer-role match is case-sensitive exact equality.** `lastImplementerModel()` matches only `Implementation` / `TDD Implementation`; a run recorded as `implementation` (append-run `--role` is free text via strFlag, no enum) makes the check skip fail-open with a misleading "no Implementation run found" info line. Exact equality correctly avoids the "Implementation Preparation" prefix collision. Case-insensitive compare would be more robust.

**I3 — "Last implementer model" keeps the last non-empty `model_id` across impl runs**, not literally the last impl run's value. Divergence is only reachable via hand-edited STATE (`model_id: null` — append-run requires `--model`), and diverges in the safe direction (check runs instead of skipping).

**I4 — Opt-in bypass surface (spec-accepted).** Even under `independence: enforce`, omitting `--model` — or passing a `--ref` that mismatches the metrics key (set-validation `--ref` uses strFlag, not refFlag — pre-existing) — skips the check with an info line. Spec D2 mandates these skip paths ("never block honest work"); the real teeth are the MODEL contract field in the prompts. Recorded so the residual is conscious.

**I5 — No production price resolver (spec-accepted).** `lookup_rules` are implemented verbatim only in tests/skills/test-aai-pricing.sh; production resolution is LLM-side per METRICS_FLUSH.prompt.md. The spec's Seam analysis explicitly accepts this ("first real flush after this change should be operator-observed").

Prior main-side ledger lines show 1–3 failures on 15 suites, so these are pre-existing, not introduced here (the +1 total is the new pricing suite, which passes).
