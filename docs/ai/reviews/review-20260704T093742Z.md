1. `reset-block` on `notes: |-` literal block → prior note lines DELETED (see W1).
2. `append-run` into `metrics: {}` inline header → structurally invalid YAML written with exit 0; `check-state.mjs` exits 0 on the broken file; `yaml.safe_load` rejects it (see W2).
3. `set-validation --ref "CHANGE-1: bad"` → `ref_id: CHANGE-1: bad` written; invalid YAML, exit 0, check-state blind to it (see W3).
4. Two concurrent mutators on one STATE → one committed mutation silently lost (last rename wins) (see W4).
5. Misspelled flags (`--evidnce`, `--notse`) → silently ignored, evidence/notes dropped, exit 0 (see W5).
6. `status: fail  # comment` → `reset-block` safely REFUSES exit 2 ("unexpected status — repair STATE first"). Good conservative behavior.
7. CRLF STATE file → mutation succeeds, whole file rewritten LF (documented normalization; see I2).
8. Duplicate-key STATE + `log-tick` → tolerated by design; reads FIRST duplicate block (see I3).
9. Folded-scalar hostile content (`metrics:` at column 0 inside a note, trailing `#`): NOT reproducible — `textFieldLines` trims and re-indents every segment by construction (state.mjs:277-285), so scalar content can never reach column 0. Safe.

### WARNING (should fix before merge / as follow-up)

**W1 — reset-block destroys `|-`/`|+`/`>+` style notes (audit-history data loss).**
`.aai/scripts/state.mjs:736-744`: the marker-append logic special-cases `>-`, `|`, `>`
but not `|-` (or chomping variants). For `notes: |-` the else-branch
`bl.splice(idx, n, '  notes: >-', rest, marker)` replaces the ENTIRE field span,
deleting all prior note content lines and leaving the literal string `|-` as the
note body. Reproduced: two-line `|-` note reduced to `>-\n    |-\n    reset by...`.
Violates D6's "appends a reset marker to its notes" intent (prior notes are audit
history). Mitigation: the CLI itself only ever emits `>-`, and the real
`docs/ai/STATE.yaml` uses only `>-` today — reachable only via hand-edited/legacy
files, hence WARNING not ERROR. Fix: treat any header matching `/^[>|][+-]?$/` as
block-scalar and append after the span.

**W2 — engine writes structurally invalid YAML when a target block header carries an inline value; the paired validator cannot see it.**
`findBlock`/`editBlock` (`.aai/scripts/state.mjs:215-249`) treat `metrics: {}` (or
`current_focus: {}`, `metrics: null`) as a block header and splice nested lines
under it. Reproduced: `metrics: {}` + `append-run` → `metrics: {}` followed by
`  work_items:` — invalid YAML (mapping value given twice), exit 0, and
`check-state.mjs` exits 0 (it only checks duplicate top-level keys). The engine
guards the analogous inline forms it knows (`agent_runs: []` conversion at :675,
non-empty inline refusal at :682, `work_items: []` at :634, `active_work_items: []`
at :396) but not an inline top-level block header. This is the one seam where the
"writer never produces what the validator rejects" guarantee (SEAM-1) is satisfied
only because the validator is weak. Not reachable from CLI-authored files
(ensureBlock always writes bare `key:` headers) — hostile/legacy input only.
Fix: in `editBlock`, refuse (exit 2) when the located header line has a non-empty
inline value other than `[]` handled conversions.

**W3 — plain-scalar values written verbatim: a value containing `: ` (or leading `#`/`[`/`{`) produces invalid YAML with exit 0.**
`scalarLine` (`.aai/scripts/state.mjs:273-275`) never quotes/validates. Free-text
fields (`--notes`, `--rationale`, `--scope`, `--question`) are safe via `>-`
block scalars, but `set-validation --ref`, `set-worktree --branch/--base-ref/--path`,
`append-run --model`, `set-tdd-cycle --test-id/--*-path` are unvalidated plain
scalars. Reproduced with `--ref "CHANGE-1: bad"`. check-state cannot detect it.
LLM callers are exactly the population that will one day pass a colon-bearing
string. Fix: reject (exit 2) scalar values matching `/:\s|^[#\[{&*!|>%@`"']/`
or write them single-quoted.

**W4 — concurrent invocations: lost update; no lock claimed, posture undocumented.**
Reproduced: two parallel `state.mjs` mutators on one file → both exit 0, one
mutation silently absent (read-modify-write race; rename-last wins). The repo's
protection is prompt-level only (`.aai/SUBAGENT_PROTOCOL.md:83-87` sole-writer
rule; ORCHESTRATION_PARALLEL keeps the orchestrator as single writer), but
neither SPEC-0012 nor the `state.mjs` header states the concurrency contract
(last-writer-wins, single-writer assumed), and the CLI neither takes the
RFC-0004 docs-lock nor detects a mid-flight change (e.g. mtime/content recheck
before rename). The migrated role prompts unconditionally instruct each role to
run `append-run` itself, which under parallel dispatch conflicts with the
sole-writer rule unless the subagent honors SUBAGENT_PROTOCOL (pre-existing
tension, now mechanized). Fix: document last-writer-wins + single-writer
assumption in the state.mjs header and SPEC-0012, or add an optimistic pre-rename
content recheck.

**W5 — unknown/misspelled flags are silently ignored (data silently dropped).**
`parseArgs` (`.aai/scripts/state.mjs:92-112`) collects any `--flag`; subcommands
read only the keys they know; leftovers are never diagnosed. Reproduced:
`set-validation --status pass --evidnce tests/a.log --notse "..."` → exit 0,
evidence and notes silently absent. For a closed-set CLI built specifically so
LLM mistakes fail loud (D1 posture, mirrored from append-event.mjs), a typoed
flag is the most likely LLM mistake and it currently fails silent. Fix: per
subcommand, reject unconsumed flags (exit 2).

### INFO (nice to fix)

**I1** — `set-focus` never touches `current_focus.spec_path` unless `--spec-path`
is passed, so switching focus leaves the PREVIOUS scope's spec_path behind
(reproduced). No consumer of `current_focus.spec_path` found in prompts/scripts,
so staleness is latent only. Consider nulling it when not supplied (as done for
ref/path with `--type none`).

**I2** — CRLF input file: mutation rewrites the whole file LF (spec D2/edge-case
documents "normalize on read"), so TEST-020's diff-locality guarantee does not
hold for CRLF checkouts and git will show a full-file diff. Acceptable/documented;
noting for operators on Windows checkouts (this repo's TECHNOLOGY.md is itself a
CRLF artifact).

**I3** — On a duplicate-key STATE, `log-tick` (tolerated by design) reads the
FIRST duplicate block via `findBlock`, while YAML last-key-wins semantics would
give the LAST — `focus/validation *_after` fields in the tick line can reflect
the dropped block. Corrupt-input-only skew.

**I4** — `--state` given WITHOUT a value (trailing flag) is `true` and silently
falls back to the default `docs/ai/STATE.yaml` (`state.mjs:815`): a fixture-
targeting invocation that drops its argument mutates the REAL runtime state with
exit 0. Consider failing on a non-string `--state`/`--ticks`.

**I5** — METRICS_FLUSH step 5d primary path uses `set-validation --status not_run`,
which self-stamps `run_at_utc` (never null), so the CLI path cannot reach the
documented default shape (`run_at_utc: null`) without the follow-up hand edit the
prompt prescribes. Acknowledged in-prompt; a `--reset` mode would remove the
residual hand edit.

**I6** — Filesystem failures during write (EACCES on dir, ENOSPC) surface as
uncaught Node exceptions with exit code 1, colliding numerically with the
"integrity refusal" exit-1 contract (message text disambiguates; target file is
still never corrupted because the tmp+rename ordering holds).

**I7** — `intFlag` accepts negative values (`--tokens-in -5`, `--tick -1`);
stale `*.tmp-<pid>` files from crashed runs are never cleaned up (TEST-005 calls
them "ignorable"); a new `set-phase` item field inserted at the end of a block
lands after the trailing blank line (valid YAML, cosmetically off).
