## Targeted defect analysis (dispatch SPECIFIC ATTENTION — judged independently)

1. **Multiset working-vs-HEAD diff (`computeAddedFlags`, L161-174)** — CORRECT.
   HEAD lines counted into a multiset map; each working line consumes one HEAD
   occurrence (protected) else flags added. A line committed once AND re-added
   uncommitted → first occurrence consumed/protected, second flagged added →
   exactly the stray copy harvested. A committed line present in equal count is
   never harvested; an added line (absent from HEAD) is never missed. Exact-line
   keys (not trimmed) — correct for JSONL.
2. **Carry-before-clean ordering** — CORRECT and unconditionally gated. In
   `processFile` a failed/`!ok` write returns BEFORE any cleanup (L300); a failed
   `git add` returns before cleanup (L304). `cleanupSource` removes a source line
   ONLY when `carriedFlags[i] && destPresenceSet.has(line.trim())` (L273), and
   `destPresenceSet` is re-read from the ACTUAL destination file after the write
   (L310-311). No path removes a source line not confirmed in dest. Metrics is
   fully processed before events; an events-write failure exits 1 with metrics
   already safely in dest + staged and events source untouched.
3. **Surgical source scrub (`cleanupSource`, L269-284)** — CORRECT. Position-
   accurate via the parallel `carriedFlags` array from the harvest snapshot, not
   string re-matching, so an identical committed (unflagged) line at an adjacent
   index is kept; only the flagged, dest-confirmed occurrence is dropped.
4. **`writeAndVerify` revert (L240-264)** — byte-exact. On verify mismatch it
   restores the original `existingContent` (or `rmSync` if the dest was absent).
   `git add` runs only after `ok:true`, so a reverted/aborted write is never
   staged — no orphaned partial append. Best-effort revert failure still returns
   `ok:false` → exit 1, source untouched (fail-closed).
5. **Ref filter (L176-188)** — no over-broad match. `metricsMatches` uses
   `split('/').includes(ref)` (whole-segment, so `foo` ≠ `foobar`);
   `eventsMatches` uses `=== ref || startsWith(`${ref}/`)` (so `foo` matches
   `foo/x` but not `foobar/x`). Asymmetry is spec-sanctioned (flush refMatches
   vs append-event PARENT/suffix rollup).
6. **SKILL_PR step 2b** — inserted after step 2 (STAGE), before step 3 (AUDIT);
   exit 1 STOPs, exit 0 proceeds; METRICS/EVENTS added to expected-companions;
   FALLBACK note present. Matches spec Implementation plan exactly.
7. **Fixture safety** — every test runs with `cwd` = a mktemp fixture tree; real
   ledgers md5-unchanged (verified).
