## Verdict 2 — code_quality: PASS (2 NON-BLOCKING findings)

### NB-1 — `--phase2 --resync` cannot heal an old-shape canonical doc (docs-canon-core.mjs:468-477)
Details and reproduction in the YAML block. The skip guard tests only source
hashes; a shape-stale canonical with unchanged sources is skipped forever, so
the migration path named in docs-model.mjs, CANONICAL_TEMPLATE.md:25 and spec
D3 does not work. No gate consumes validateSectionContract yet and
docs/canonical/ is empty in this repo, so nothing breaks today — but stage 2
is precisely when this becomes load-bearing.
**Recommended disposition:** promote to a tracked follow-up ref (must land
before/with RFC-0011 stage 2), or remediate-in-tree if the orchestrator
prefers: extend the skip guard to also re-render when the existing canonical
fails validateSectionContract (shape-drift), or correct the three doc surfaces
to an honest path.

### NB-2 — unvalidated domain slug now throws after archive mutation (docs-canon-core.mjs:314-322 render, :509-522 order, :383 preflight gap)
`renderCanonicalDoc` gained a throwing dependency (`domainToReqDomain`), and
the fail-fast preflight `validatePhase2Plan` never checks map domain keys
against `DOMAIN_SLUG_RE`. A malformed operator-edited map now crashes phase 2
between the archive move and the canonical write — the exact half-mutated
state the preflight promises to prevent.
**Recommended disposition:** remediate-in-tree (add a per-domain
DOMAIN_SLUG_RE check to validatePhase2Plan + one test case; ~3 lines), or fold
into the same follow-up ref as NB-1.
