## Stage 2 — Code Quality (focused on the false-negative attack surface)

### ERROR E1 — `normalizePath` does not canonicalize; whole-repo `"."` and several alias forms are dangerous FALSE NEGATIVES
**File:** `.aai/scripts/orchestration-mode.mjs:67-86` (`normalizePath`), consumed by `pathsOverlap` (115-118) and `isUncertain` (103-111).

`normalizePath` only trims, strips one trailing `/`, and drops a trailing `/**`|`/*`. It does NOT: collapse a leading `./`, collapse interior `//`, resolve `..` segments, fold case, nor treat `"."` (whole repo) as covering everything. Each yields two **genuinely-overlapping** scopes judged **independent** and co-scheduled in ONE parallel group. Observed selector output (all `{"mode":"parallel","k":2,...A,B in one parallel group}`):

| # | scope A path | scope B path | reality | selector output | correct? |
|---|---|---|---|---|---|
| 1 | `"."` | `"apps/api/"` | A = whole repo ⊇ B | **parallel, A+B grouped** | NO (must conflict) |
| 2 | `"./apps/api/"` | `"apps/api/"` | same dir | **parallel, A+B grouped** | NO |
| 3 | `"apps//api/"` | `"apps/api/"` | same dir | **parallel, A+B grouped** | NO |
| 4 | `"apps/web/../api/"` | `"apps/api/"` | same dir | **parallel, A+B grouped** | NO |

Reproduce (#1):
```
printf '%s' '{"orchestration_mode":"auto","k_max":2,"locks_available":true,"scopes":[{"id":"A","role_kind":"write","review_scope_paths":["."],"isolation":"inline"},{"id":"B","role_kind":"write","review_scope_paths":["apps/api/"],"isolation":"inline"}]}' | node .aai/scripts/orchestration-mode.mjs
-> {"mode":"parallel","k":2,"groups":[{"kind":"parallel","scopes":["A","B"]}],"reasons":{}}
```

**Why this is ERROR, not cosmetic:**
- `role_kind:"write"`, `isolation:"inline"` — two write agents co-scheduled on overlapping/whole-repo paths is the precise same-file corruption that RFC-0004 locks + this selector's fail-closed gate exist to prevent. A false negative here defeats the safety floor.
- It is **reachable**: the orchestrator (SKILL_LOOP) sources `review_scope_paths` from `code_review.scope` / `worktree.inline_review_scope` / STATE affected paths. `"."` is the natural declaration for a repo-wide change/sweep; `"./src"` style and case-variant paths are common authoring forms. `normalizePath` is INSIDE the frozen core, so canonicalization is the selector's responsibility, not the caller's.
- **Inconsistency proving it's a bug, not intent:** `""`, `"*"`, `"**"`, `"*.md"`, and `"/"` ALL correctly reduce to `null` -> uncertain -> fail-closed. Only `"."` (the most common whole-repo spelling) slips through as a concrete literal that overlaps nothing.
- `orchestration_mode:"parallel"` does NOT save it (confirmed `"."` vs `apps/api/` still parallel) — the opt-in correctly cannot bypass overlap, but the overlap test itself is blind here.

**Suggested fix (selector-side, do not implement here):** in `normalizePath`, strip leading `./`, collapse `//`, and reject (`-> null`/uncertain) any path that resolves to `.`/empty or contains a `.`/`..` segment; optionally case-fold when targeting case-insensitive FS. Add overlap tests for `.`,`./x`,`a//b`,`a/b/../c`,`Case/Var` to the corpus (these are exactly the shapes the 9 validator shapes + 17 tests missed).
