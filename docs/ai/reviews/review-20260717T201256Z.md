## Code_quality â€” findings with dispositions

Independent judgement on the five SPECIFIC-ATTENTION points:

1. **Masking pass complexity / off-by-one.** `computeConsumedLineMask` is a single
   O(n) pass: the outer loop advances `i` to `nextIndex` (past each consumed span),
   so no line is re-entered; adjacent blocks do not re-scan; env files do not nest.
   No quadratic blow-up in file size. The closing line index (`nextIndex-1`) is
   correctly marked consumed and the assignment's own start line is correctly left
   un-masked â€” no off-by-one in the normal (no interior quote) case. The mask and
   the target-key scan share `consumeQuotedValue`, so their continuation
   boundaries are always consistent (no risk of the scan finding a value the mask
   thought interior).
2. **Never-echo on ERROR/malformed paths.** No path in the new code interpolates
   value content. `consumeQuotedValue` builds the multiline `value` string but only
   returns it internally for a `.length` test; `refLabel`/notes/exit strings carry
   only references (name/path/key) or fixed strings. Unterminated-to-EOF does not
   throw; the top-level try/catch still guards everything with the fixed exit-1
   string. Confirmed by probe D (sentinel grep=0).
3. **RR-2 (unterminated/embedded quote).** See NON-BLOCKING finding above.
   Reproduced in the dangerous direction (missing -> `exists`, probes A/B). It is
   the spec-frozen accepted limitation (SPEC-0049 Seam analysis RR-2) and never a
   byte leak. Disposition: **accepted-limitation, already recorded in the frozen
   spec + decisions.jsonl (CHANGE-0034 NB-1 lineage)** â€” no new artifact required.
   My independent judgement matches the spec's: not blocking, because (a) it is a
   strict improvement over the pre-fix parser, (b) never-echo is preserved, and (c)
   the residual dangerous case requires both a stray/escaped quote in an interior
   line AND a following column-0 `KEY=` line. RECOMMENDATION for the orchestrator:
   a follow-up ISSUE to make the ambiguous-close case fail toward `missing`
   (conservative/safe) would close the last dangerous-direction gap â€” optional
   hardening, not a merge blocker.
4. **Quote symmetry / escaped quote.** `'` and `"` are handled symmetrically
   (open-char detection, close-char detection, and strip all key on the same
   captured quote char). An escaped `\"` is NOT special-cased â€” it counts as a
   close (under-mask) â€” same RR-2 limitation as (3), confirmed by probe B.
5. **CRLF.** `raw.split(/\r?\n/)` strips the `\r`, so a trailing `\r` never defeats
   the `lines[j].includes(quote)` close detection or the `closedSameLine` last-char
   test. Confirmed by probe C (INNER -> missing, REAL -> exists, CERT -> exists).

No BLOCKING findings. One NON-BLOCKING (RR-2, accepted-limitation disposition), one
