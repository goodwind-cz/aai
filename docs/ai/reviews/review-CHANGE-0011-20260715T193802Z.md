### Digest-sufficiency analysis (highest-risk spot) — SAFE, with field comparison

ORCHESTRATION.prompt.md STATE DISCOVERY (lines 49–64) needs, for its 14-rule logic:
STATE presence/validity; project_status; human_input.required; workflow normalized;
TECHNOLOGY.md currency; requirements→spec mapping; SPEC-FROZEN; implementation_strategy;
worktree.recommendation + user_decision; unvalidated implementation; code_review
required/status/outdatedness; latest validation verdict.

loop-digest.mjs --json emits (reviewer-executed, exact): ticks, durationSeconds,
harnessVersion, startedUtc, endedUtc, scopes[], finalValidation, recoveries,
recoveryOutcomes[], stopReason, cost{input,output,cacheRead,usd,any},
git{branch,uncommitted,recentCommits[]}.

Overlap is nearly empty: human_input, project_status, worktree decision,
implementation_strategy, and code_review status are ALL absent from the digest. The
digest would therefore be INSUFFICIENT as the sole state payload — but the change does
not make it the sole source. SKILL_LOOP.prompt.md:213–217 explicitly instructs "Do NOT
inject full docs/ai/STATE.yaml contents: the orchestrator prompt's step 1 already reads
docs/ai/STATE.yaml from disk itself (the authoritative source)", and both orchestrator
prompts independently mandate the disk read: ORCHESTRATION.prompt.md:8 (REQUIRED
CAPABILITIES "Read and update docs/ai/STATE.yaml"), :12 (first AUTHORITATIVE SOURCE),
:49 ("STATE DISCOVERY (MANDATORY)"); ORCHESTRATION_PARALLEL.prompt.md:8 likewise. The
orchestrator also MUST update STATE.yaml before stopping (ORCHESTRATION:167), so an
orchestrator without file access was already non-functional pre-change — the payload
switch removes no capability any conforming dispatch ever relied on. The digest is
trajectory context (recoveries, stagnation, cost), not decision state. The DEGRADATION
clause (script/node failure → inject full STATE.yaml, note in tick line) closes the
script-absent case. Documented field list matches script reality character-for-character.

- N1 — STATE_FALLBACK.md:3 header overclaims "read ONLY when .aai/scripts/state.mjs is
  absent", but two prompts now reference the file on the PRIMARY path:
  METRICS_FLUSH.prompt.md:58–59 ("see the flush-reset defaults in .aai/STATE_FALLBACK.md"
  while hand-nulling residual fields after the CLI resets) and
  SKILL_CHECK_STATE.prompt.md:90–91 ("full write-safety rules: .aai/STATE_FALLBACK.md").
