# Bootstrap Skill - Project Architecture Detection

## Goal
Automatically detect the project's architecture and generate optimized, project-specific Claude Code skills that save tokens and improve efficiency.

## Instructions

### 1. Detect Project Architecture

Scan the project root and analyze:

**Package Managers:**
- `package.json` → Node.js/npm/yarn/pnpm
- `requirements.txt`, `pyproject.toml`, `poetry.lock` → Python
- `Cargo.toml` → Rust
- `go.mod` → Go
- `pom.xml`, `build.gradle` → Java/Maven/Gradle

**Test Frameworks:**
- `playwright.config.{ts,js}` → Playwright
- `jest.config.{ts,js}` → Jest
- `vitest.config.{ts,js}` → Vitest
- `pytest.ini`, `tests/conftest.py` → Pytest
- `cypress.json` → Cypress

**Build Tools:**
- `vite.config.{ts,js}` → Vite
- `webpack.config.js` → Webpack
- `tsconfig.json` → TypeScript
- `Dockerfile` → Docker
- `.github/workflows/` → GitHub Actions

**Backend Frameworks:**
- `app.py`, `main.py` with FastAPI/Flask imports → Python web
- `server.ts`, `index.ts` with Express/Fastify → Node.js web
- Files in `src/` with Next.js/NestJS patterns

**MCP Servers:**
- Check if Playwright MCP server is available via `mcp list` command
- Check for other MCP servers (filesystem, github, etc.)

### 2. Generate Project-Specific Skills

Create `.claude/skills.local/` directory (ignored by ai-os-sync) and generate:

#### A. Test Runner Skill

**If Playwright detected:**

Create `.claude/skills.local/aai-test-e2e/SKILL.md`:
```markdown
---
name: aai-test-e2e
description: Run Playwright E2E tests efficiently. Auto-detects MCP server vs local execution.
---

# E2E Test Runner

## Strategy Selection

1. **Check for Playwright MCP server:**
   - Run: `mcp list` and check if playwright server is available
   - If available: Use MCP server for browser automation (more efficient)
   - If not: Use local Playwright CLI

2. **Execute tests:**

**With MCP (preferred):**
```bash
# Use playwright MCP server commands
mcp playwright run-tests [test-pattern]
```

**Without MCP (fallback):**
```bash
# Use local Playwright
npx playwright test [test-pattern]
```

## Token Optimization

- **Single test:** `npx playwright test path/to/test.spec.ts`
- **Specific suite:** `npx playwright test --grep "@smoke"`
- **Headed mode for debugging:** `npx playwright test --headed`
- **Show report:** `npx playwright show-report`

## Common Patterns

- `test-e2e` → Run all E2E tests
- `test-e2e login` → Run tests matching "login"
- `test-e2e --headed` → Run with visible browser
```

**If Jest/Vitest detected:**

Create `.claude/skills.local/aai-test-unit/SKILL.md`:
```markdown
---
name: aai-test-unit
description: Run unit tests efficiently with Jest/Vitest.
---

# Unit Test Runner

**Detected framework:** [Jest|Vitest]

## Commands

- **All tests:** `npm test`
- **Watch mode:** `npm test -- --watch`
- **Coverage:** `npm test -- --coverage`
- **Specific file:** `npm test -- path/to/test`
- **Update snapshots:** `npm test -- -u`

## Token Optimization

Only run tests related to changed files:
```bash
npm test -- --onlyChanged
```
```

#### B. Build Skill

Create `.claude/skills.local/aai-build/SKILL.md`:
```markdown
---
name: aai-build
description: Build the project efficiently based on detected build tool.
---

# Build Command

**Detected tool:** [Vite|Webpack|tsc|other]

## Commands

- **Development build:** `npm run dev`
- **Production build:** `npm run build`
- **Type check only:** `npx tsc --noEmit`

## Token Optimization

For quick validation without full build:
```bash
# Type check only (fast)
npx tsc --noEmit

# Build specific entry point
npm run build -- --mode=production
```
```

#### C. Lint/Format Skill

Create `.claude/skills.local/aai-lint/SKILL.md`:
```markdown
---
name: aai-lint
description: Run linters and formatters efficiently.
---

# Lint & Format

**Detected tools:** [ESLint, Prettier, Black, Ruff, etc.]

## Commands

- **Lint only:** `npm run lint`
- **Lint + fix:** `npm run lint -- --fix`
- **Format:** `npm run format`

## Token Optimization

Lint only changed files:
```bash
# ESLint on specific files
npx eslint src/components/Button.tsx --fix

# Prettier on specific files
npx prettier --write src/components/Button.tsx
```
```

#### D. Deploy Skill (if CI/CD detected)

Create `.claude/skills.local/aai-deploy/SKILL.md` based on GitHub Actions, Docker, or other deployment methods.

### 3. Create Skills Index

Create `.claude/skills.local/README.md`:
```markdown
# Project-Specific Skills

Auto-generated by `/aai-bootstrap` on [DATE]

## Available Skills

- `/aai-test-e2e` - Run Playwright E2E tests
- `/aai-test-unit` - Run Jest/Vitest unit tests
- `/aai-build` - Build the project
- `/aai-lint` - Lint and format code
- `/aai-deploy` - Deploy to staging/production

## Architecture Detected

- **Language:** [TypeScript/Python/etc]
- **Framework:** [Next.js/FastAPI/etc]
- **Test Framework:** [Playwright/Jest/etc]
- **Build Tool:** [Vite/Webpack/etc]
- **Package Manager:** [npm/yarn/pnpm/poetry/etc]

## Maintenance

Run `/aai-bootstrap` again to regenerate skills after architecture changes.

**Note:** This directory (`.claude/skills.local/`) is preserved during `ai-os-sync` updates.
```

### 4. Update .gitignore

Ensure `.claude/skills.local/` is NOT ignored (it's project-specific, should be committed).

But add to `.gitignore`:
```
.claude/skills.local/.cache
```

Output a summary:

- aai-lint - ESLint + Prettier

Try: /aai-test-e2e, /aai-test-unit, /aai-build, /aai-lint
```

## Token Optimization Strategies

1. **Lazy Loading:** Skills only read config files when invoked
2. **Smart Defaults:** Pre-configure common patterns in skills
3. **Incremental Testing:** Run only affected tests
4. **Cache Awareness:** Use build caches, don't rebuild unnecessarily
5. **MCP Preference:** Use MCP servers when available (offload work from agent)

## Safety

- Never overwrite existing `.claude/skills.local/` skills without user confirmation
- Always validate generated commands before execution
- Log all generated skills to skill index for transparency
