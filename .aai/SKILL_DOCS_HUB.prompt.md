# Docs Hub Skill — Interactive AAI Skills Documentation

## Goal
Generate a beautiful, interactive HTML documentation catalog of all AAI skills, with search, filtering, categories, relationships, and usage examples. Publishable via `/aai-share`.

## Instructions

### Step 1: Discover All Skills

Scan for AAI skills in the project:

```bash
find .claude/skills -name "SKILL.md" -type f | sort
```

For each skill found, extract:
- Skill name (from YAML frontmatter `name:`)
- Description (from YAML frontmatter `description:`)
- Corresponding prompt file path (`.aai/<SKILL_NAME>.prompt.md`)

### Step 2: Read Skill Metadata

For each skill, read:
1. **SKILL.md** - Extract `name` and `description` from YAML frontmatter
2. **Corresponding .prompt.md** - Extract:
   - Goal section
   - Prerequisites (if any)
   - Instructions overview
   - Key sections/phases
   - Dependencies on other skills (mentioned skill names)
   - When to use / when not to use (if mentioned)
   - Example usage (if provided)

### Step 3: Categorize Skills

Automatically categorize skills based on their purpose:

- **Intake & Planning**: Skills that handle requirements, planning, specs
  - Examples: `aai-intake`, `aai-check-state`

- **Development & Testing**: Skills for TDD, implementation, validation
  - Examples: `aai-tdd`, `aai-validate-report`, `aai-test-skills`

- **Workflow & Orchestration**: Skills that manage workflow and agent coordination
  - Examples: `aai-loop`, `aai-hitl`, `aai-worktree`

- **Quality & Maintenance**: Skills for canonicalization, flushing, cleanup
  - Examples: `aai-canonicalize`, `aai-flush`

- **Publishing & Sharing**: Skills for documentation and sharing
  - Examples: `aai-share`, `aai-bootstrap`

Use heuristics from skill names and descriptions to auto-categorize.

### Step 4: Extract Relationships

For each skill, identify:
- **Prerequisites**: Other skills that should run before this one
- **Follows**: Skills commonly run after this one
- **Related**: Skills in similar workflows

Look for skill names mentioned in prompts (e.g., "run `/aai-intake` first", "after `/aai-tdd`").

### Step 7: Implement JavaScript Features

**Search:**
```javascript
function filterSkills(searchTerm, category) {
    const cards = document.querySelectorAll('.skill-card');
    cards.forEach(card => {
        const name = card.querySelector('h3').textContent.toLowerCase();
        const desc = card.querySelector('.skill-description').textContent.toLowerCase();
        const matchesSearch = name.includes(searchTerm) || desc.includes(searchTerm);
        const matchesCategory = category === 'all' || card.dataset.category === category;
        card.style.display = (matchesSearch && matchesCategory) ? 'block' : 'none';
    });
}
```

**Theme Toggle:**
```javascript
function toggleTheme() {
    document.body.classList.toggle('dark-theme');
    localStorage.setItem('theme', document.body.classList.contains('dark-theme') ? 'dark' : 'light');
}
```

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Skills not found | Check `.claude/skills/` structure |
| Prompt files missing | Verify `.aai/*.prompt.md` naming convention |
| Relationships unclear | Manually review prompt files for skill mentions |
| Categories wrong | Adjust categorization heuristics in Step 3 |

## Maintenance

Re-run `/aai-docs-hub` whenever:
- New skills are added
- Skill descriptions change
- Prompt files are updated
- You want to refresh the catalog

The hub is fully regenerated each time for consistency.
